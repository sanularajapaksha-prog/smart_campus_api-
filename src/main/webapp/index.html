Smart Campus API Testing Guide

HTML test reference for your Smart Campus Sensor & Room Management API.

Project Overview

This Smart Campus API is currently running on Apache Tomcat and is accessible through the base URL shown below. The system was developed as part of the Client-Server Architectures coursework using JAX-RS.

The API simulates a campus environment where:

  • Rooms can be created and managed
  • Sensors can be assigned to rooms
  • Sensor readings can be recorded and retrieved

All data is stored in memory (HashMap / ArrayList) as required by the coursework, so data will reset when the server restarts.

This API follows REST principles including:

  • Resource-based endpoints
  • Proper HTTP methods (GET, POST, DELETE)
  • Structured JSON responses
  • Error handling with correct status codes (403, 409, 422, 500)

Status: Server is running and ready for testing.

Base URL

http://localhost:8080/api/v1
Use this base URL in Postman or with curl. WAR is deployed as ROOT context on Tomcat 9.

1. Discovery Endpoint

GET/api/v1/

Request

GET http://localhost:8080/api/v1/

Expected Example Response

{
  "message": "Smart Campus API",
  "version": "v1"
}

2. Create Room

POST/api/v1/rooms

Raw JSON Body

{
  "id": "R1",
  "name": "Lab 1",
  "capacity": 40
}

curl

curl -X POST http://localhost:8080/api/v1/rooms ^
-H "Content-Type: application/json" ^
-d "{\"id\":\"R1\",\"name\":\"Lab 1\",\"capacity\":40}"

3. Get All Rooms

GET/api/v1/rooms

Request

GET http://localhost:8080/api/v1/rooms

Expected Example Response

[
  {
    "id": "R1",
    "name": "Lab 1",
    "capacity": 40,
    "sensorIds": []
  }
]

4. Get Single Room

GET/api/v1/rooms/R1

GET http://localhost:8080/api/v1/rooms/R1

5. Create Sensor (Valid)

POST/api/v1/sensors

Raw JSON Body

{
  "id": "S1",
  "type": "Temperature",
  "status": "ACTIVE",
  "currentValue": 22.5,
  "roomId": "R1"
}

curl

curl -X POST http://localhost:8080/api/v1/sensors ^
-H "Content-Type: application/json" ^
-d "{\"id\":\"S1\",\"type\":\"Temperature\",\"status\":\"ACTIVE\",\"currentValue\":22.5,\"roomId\":\"R1\"}"

6. Create Sensor with Invalid Room (422 / 400 test)

POST/api/v1/sensors

Raw JSON Body

{
  "id": "S2",
  "type": "CO2",
  "status": "ACTIVE",
  "currentValue": 30,
  "roomId": "INVALID"
}

Expected Example Error

{
  "status": 422,
  "error": "Unprocessable Entity",
  "message": "Room does not exist"
}

7. Get All Sensors

GET/api/v1/sensors

GET http://localhost:8080/api/v1/sensors

8. Filter Sensors by Type

GET/api/v1/sensors?type=Temperature

GET http://localhost:8080/api/v1/sensors?type=Temperature

9. Add Sensor Reading

POST/api/v1/sensors/S1/readings

Raw JSON Body

{
  "value": 25.5
}

curl

curl -X POST http://localhost:8080/api/v1/sensors/S1/readings ^
-H "Content-Type: application/json" ^
-d "{\"value\":25.5}"

10. Get Sensor Readings

GET/api/v1/sensors/S1/readings

GET http://localhost:8080/api/v1/sensors/S1/readings

Expected Example Response

[
  {
    "id": "uuid-1",
    "timestamp": 1713780000000,
    "value": 25.5
  }
]

11. Maintenance Sensor Test (403)

POST/api/v1/sensors then /api/v1/sensors/S3/readings

Create Maintenance Sensor

{
  "id": "S3",
  "type": "Temperature",
  "status": "MAINTENANCE",
  "currentValue": 20,
  "roomId": "R1"
}

Try Adding Reading

{
  "value": 30
}

Expected Example Error

{
  "status": 403,
  "error": "Forbidden",
  "message": "Sensor is in maintenance mode"
}

12. Delete Room with Sensors Inside (409)

DELETE/api/v1/rooms/R1

DELETE http://localhost:8080/api/v1/rooms/R1

Expected Example Error

{
  "status": 409,
  "error": "Conflict",
  "message": "Room still contains sensors"
}

Content-Type: text/plain
invalid data

14. Invalid Endpoint Test

15. Global Error / Broken JSON Test

POST/api/v1/rooms

Broken JSON Body

  "id": "R2",
  "name": "Lab 2",

Expected Example Error

{
  "status": 500,
  "error": "Internal Server Error",
  "message": "Unexpected error occurred"
}

Important Test Coverage

  • 200 OK for successful GET requests
  • Successful POST requests for room, sensor, and reading creation
  • 409 Conflict when deleting a room that still contains sensors
  • 422 or 400 when creating a sensor with an invalid roomId
  • 403 Forbidden when sensor status is MAINTENANCE and a reading is added
  • Filtering using ?type=
  • Nested resource testing with /sensors/{id}/readings
